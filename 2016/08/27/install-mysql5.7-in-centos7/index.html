centos7安装mysql5.7.14

Aug 27, 2016 | technology | Hits

本次 Mysql 为Community 5.7.1.4 版本，安装方式为通用Linux安装方式，即大多数Linux平台都可以采用该方式进行安装。即通用安装方式。
(说明：mysql-VERSION-OS.tar.gz 表示当前安装的版本号)OS=linux2.6-x86_64/等 VERSION = 版本号
一、安装环境

2、下载 Mysql 、解压缩、创建软连接，与官方提供解压缩路径有些许差别。

参考 Mysql 官方指导建议进行安装，多用在Mysql学习环境，下列命令来源Mysql官方，原文参见 Installing MySQL on Unix/Linux Using Generic Binaries

a.检查是否已安装，grep的-i选项表示匹配时忽略大小写

*可见已经安装了库文件，应该先卸载，不然会出现覆盖错误。注意卸:载时使用了–nodeps选项，忽略了依赖关系,当然官网也同时提供了警告说明文档

Warning
If you have previously installed MySQL using your operating system native package management system, such as yum or apt-get, you may experience problems installing using a native binary. Make sure your previous MySQL installation has been removed entirely (using your package management system), and that any additional files, such as old versions of your data files, have also been removed. You should also check for configuration files such as /etc/my.cnf or the /etc/mysql directory and delete them.

For information about replacing third-party packages with official MySQL packages, see the related Apt guide or Yum guide.

b. 添加mysql组和mysql用户，用于设置mysql安装目录文件所有者和所属组。

*useradd -r参数表示mysql用户是系统用户，不可用于登录系统。

c. 将二进制文件解压到指定的安装目录，我们这里指定为/usr/local

加压后在/usr/local/生成了解压后的文件夹mysql-5.7.14-linux2.6-x86_64，这名字太长，我们为它建立一个符号链接mysql，方便输入。

d. /usr/local/mysql/下的目录结构

e. 进入mysql文件夹，也就是mysql所在的目录，并更改所属的组和用户。

(注意：mysql后面有个点 “ .”标识把、usr/local/mysql 下的所有文件的所有者改为mysql这个用户，同时用户组也修改为mysql这个用户组)

f. 初始化mysql的数据表格
在mysql5.7.6之前的版本，采用bin/mysql_install_db –user=mysql 这个在5.7.6之后已经不再采用。
在mysql5.7.6之后的版本，采用bin/mysqld –initialize –user=mysql

注意mysql服务进程mysqld运行时会访问data目录，所以必须由启动mysqld进程的用户（就是我们之前设置的mysql用户,mysql文件下。设置只有mysql用户有权限）执行这个脚本，或者用root执行，但是加上参数–user=mysql。

如：[root@localhost mysql]scripts/mysql_install_db –user=mysql

g. 将mysql/目录下除了data/目录的所有文件，改回root用户所有，mysql用户只需作为mysql/data/目录下所有文件的所有者。

h. 复制配置文件

i. 将mysqld服务加入开机自启动项。
*首先需要将scripts/mysql.server服务脚本复制到/etc/init.d/，并重命名为mysqld。

通过chkconfig命令将mysqld服务加入到自启动服务项中。
[root@localhost mysql]#chkconfig –add mysqld 注意服务名称mysqld就是我们将mysql.server复制到/etc/init.d/时重命名的名称。

*查看是否添加成功

j. 重启系统，mysqld就会自动启动了。
*检查是否启动

*如果不想重新启动，那可以直接手动启动。

k . 运行客户端程序mysql，在mysql/bin目录中，测试能否连接到mysqld。

如果无法登录：尝试在安装目录下执行命令：./mysqld_safe –skip-grant-tables &
