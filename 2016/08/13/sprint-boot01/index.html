sprint-boot简介

Aug 13, 2016 | technology | Hits

Spring Boot

image:https://build.spring.io/plugins/servlet/buildStatusImage/BOOT-PUB["Build Status”, link=”https://build.spring.io/browse/BOOT-PUB“] image:https://badges.gitter.im/Join Chat.svg[“Chat”,link=”https://gitter.im/spring-projects/spring-boot?utm_source=badge&utm_medium=badge&utm_campaign=pr-badge&utm_content=badge“]
:docs: http://docs.spring.io/spring-boot/docs/current-SNAPSHOT/reference

Spring Boot 能让我们很容易的创建一个以spring 驱动的应用，生产级应用，服务应用的小足以让你大惊小怪。他的实现是继承spring 平台的理念，所以能让现有的spring 用户可以快速的得到他们的所需。

你可以用spring boot 创建一个标准独立的 java 应用，它可以使用java.jar 或更传统的war 包发布。此外，spring boot 还提供了命令行工具来运行spring 脚本。

Spring Boot 的最初目标：

  • 从根本上提供一个给spring 开发一个更加快速更加广泛的spring体验。
  • 坚持于开箱即用，但是让需求开始变更时，能快速的响应。
  • 提供一系列的非功能特性,在大型类项目是常见的。(如嵌入式服务器、安全指标、健康检查、外部化配置)
  • 绝对没有代码生成和XML配置要求

Modules

这里有些Spring Boot 的模块，你能在这里快速的预览：

spring-boot

Spring Boot的主要的库，他提供了功能支持Spring Boot 的其他部分的运作。
包括：

  • SpringApplication 类，提供静态的简便方法，让你能快速的开始去编写一个标准独立的Spring Application。他能创建并刷新ApplicationContext上下文的属性配置。
  • 嵌入 web 应用容器，你可以选择Tomcat 或jetty
  • 外部化配置支持
  • 方便ApplicationContext的初始化,包括支持合理的日志记录

spring-boot-autoconfigure

在classpath内容,可以配置大部分常见的应用程序。

Auto-configuration 尝试减小用户使用的beans.

spring-boot-starters

Starters 能包含一些列你的应用需要依赖的描述。你会得到一个一站式的使用spring 和其他相关技术。不需要通过查询示例代码和复制粘贴的
依赖描述符。

spring-boot-cli

Spring 命令行应用程序编译和运行Groovy源代码。这使得非常容易编写代码的绝对最小运行的应用程序。Spring CLI 还可以监控文件，当文件改变时，会自动重编译，重启。

spring-boot-actuator

Spring Boot Actuator 提供了额外的装饰你的自动applicationwith特性使它立即部署和支持生产。例如如果您正在编写一个JSON web服务那么它将提供一个服务器,安全、日志记录、外部化配置,管理端点、审计抽象,和更多。如果你想关掉内置的功能,或扩展或替换他们,这很简单。

spring-boot-loader

Spring Boot 提供了秘密武器,允许您构建一个jar文件,可以使用java jar。Generally you will not need to use
spring-boot-loader directly, but instead work with the
link:spring-boot-tools/spring-boot-gradle-plugin[Gradle] or
link:spring-boot-tools/spring-boot-maven-plugin[Maven] plugin.
